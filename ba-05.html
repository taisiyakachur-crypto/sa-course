🔵 BPMN 2.0 — основні елементи
⭕

Events (події)

Start (зелений), End (червоний), Intermediate. Що запускає процес і що його завершує.

▭

Tasks (задачі)

Прямокутники з заокругленими кутами. Людська задача, скрипт, сервіс, виклик API.

◇

Gateways (шлюзи)

Ромби для розгалуження. Exclusive (XOR), Parallel (AND), Inclusive (OR) gateway.

↔

Sequence Flows

Стрілки що з'єднують елементи в межах одного Pool. Визначають порядок виконання.

🏊

Pools & Lanes

Pool — учасник процесу (компанія, система). Lane — роль або підрозділ всередині Pool.

📨

Message Flows

Пунктирні стрілки між різними Pools. Показують обмін повідомленнями між учасниками.

📊 DFD — Data Flow Diagram

DFD показує як дані рухаються через систему. Рівні деталізації: Level 0 (Context) → Level 1 → Level 2.

🏗 C4 Model — чотири рівні архітектури
C1

Context — Контекст

Система в оточенні: хто нею користується і з якими зовнішніми системами взаємодіє. Для бізнес-стейкхолдерів.

C2

Container — Контейнери

Які великі блоки є всередині системи: веб-застосунок, мобільний додаток, API, база даних. Для команди.

C3

Component — Компоненти

Деталізація одного контейнера: які модулі, сервіси, контролери. Для розробників.

C4

Code — Код

UML Class Diagram рівня коду. Зазвичай генерується автоматично з коду. Для senior розробників.
