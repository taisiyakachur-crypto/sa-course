Архітектура систем:
шари, типи та взаємодія

Розуміння архітектури дозволяє BA описувати вимоги з урахуванням технічних обмежень і грамотно спілкуватися з архітекторами та розробниками.

🧱 Що таке архітектура системи?

Архітектура системи — це структуроване уявлення про всі основні компоненти інформаційної системи, їхню взаємодію, потоки даних, технології та правила. Вона відповідає на три питання:

🔍

З чого складається?

Перелік компонентів: UI, API, сервіси, бази даних, інтеграції, інфраструктура.

🔗

Як вони працюють разом?

Потоки даних, протоколи, синхронні та асинхронні взаємодії між компонентами.

🚀

Як забезпечити якість?

Масштабованість, безпека, надійність, продуктивність — NFR вимоги.

🔧 Компоненти системи — що BA має знати
🖥

UI/UX

Інтерфейс користувача. BA описує: wireframes, user flows, responsive вимоги, accessibility (WCAG).

🔌

API

Механізм доступу до логіки. BA описує: endpoints, параметри, response формат, статус-коди, помилки.

⚙️

Сервіси

Бізнес-логіка. BA описує: правила обробки, валідацію, бізнес-правила, flow між сервісами.

🗄

База даних

Зберігання даних. BA описує: які дані зберігати, як довго, хто має доступ, шифрування.

🔗

Інтеграції

Зовнішні системи: CRM, платіжні шлюзи, email-сервіси. BA описує: формат, частоту, помилки.

🚀

DevOps-інфра

CI/CD, моніторинг, хостинг. BA описує SLA, uptime вимоги, backup політику.

🔐

Безпека

Авторизація, аутентифікація, шифрування. BA описує ролі, дозволи, GDPR вимоги.

⚡

Кешування

Redis, Memcached для швидкості. BA описує TTL (time-to-live) та які дані можна кешувати.

📨

Черги / Events

Kafka, RabbitMQ для асинхронної обробки. BA описує: тригери, retry-логіку, порядок обробки.

⚡ Синхронні vs Асинхронні запити

🔄 Синхронний запит

Клієнт відправляє запит і чекає відповіді. Програма заблокована до отримання результату.

// Клієнт чекає відповіді сервера GET /api/user/123 → Браузер "висить" поки сервер не відповів → Отримав відповідь → продовжує
  • Простота реалізації
  • Зрозумілий потік виконання
  • Блокує програму до відповіді
  • Для: CLI, простих задач, форм

⚡ Асинхронний запит

Клієнт відправляє запит і одразу продовжує роботу. Результат обробляється пізніше.

// JS не чекає — виконується далі fetch('/api/user/123') .then(r => r.json()) .then(data => render(data)); // Цей код виконається ЗРАЗУ // не чекаючи на fetch ↑ doSomethingElse();
  • UI залишається чутливим
  • Краща продуктивність
  • Складніша логіка обробки
  • Для: веб-додатки, мікросервіси

💡 Практичний кейс: Інтеграція CRM з інтернет-магазином

🛒 Запит: "Інтегрувати CRM з інтернет-магазином"

1

Аналіз поточної архітектури

Де знаходиться CRM? Яка база даних? Який API є? BA вивчає схеми і документацію обох систем.

2

Збір вимог до інтеграції

Які дані передавати (клієнти, замовлення)? Хто ініціює синхронізацію? В який момент? Як часто?

3

Вивчення технічних можливостей

Чи є API у CRM? Який формат (JSON/XML)? Синхронна чи асинхронна інтеграція? Rate limits?

4

Опис архітектурного рішення

Діаграма потоків даних + сценарії обміну + обробка помилок + SLA для інтеграції.

5

Передача в розробку

Специфікація API (OpenAPI) + User Stories + Acceptance Criteria з прикладами даних.

🛠 Інструменти BA для роботи з архітектурою
📐

Draw.io / Lucidchart

Побудова архітектурних схем та діаграм. Безкоштовний draw.io інтегрується з Google Drive і Confluence.

🔌

Postman / Swagger

Перевірка API, знайомство з архітектурою REST. Swagger — документація та тестування в браузері.

📝

Confluence / Notion

Документування архітектурних рішень, ADR (Architecture Decision Records), схеми з поясненнями.

🌱

PlantUML / Mermaid

Графічне моделювання через текст. Sequence diagrams, C4 Model — можна зберігати в Git разом з кодом.

🗃

dbdiagram.io

ER-діаграми для моделювання структури бази даних. Простий DSL синтаксис для опису таблиць і зв'язків.

🏗

C4 Model

4-рівневий підхід до візуалізації: Context → Container → Component → Code. Зрозумілий всім стейкхолдерам.

❓ Питання для самоперевірки
  • Які три рівні в 3-tier архітектурі і що в кожному знаходиться?
  • Чим моноліт відрізняється від мікросервісів? Коли що обирати?
  • Яка різниця між синхронним та асинхронним запитом? Наведи приклад з роботи BA.
  • Для чого потрібен Kafka і чим він відрізняється від прямого API-виклику?
  • Які питання BA ставить при аналізі інтеграції двох систем?
  • Що таке C4 Model і для якої аудиторії кожний рівень?
